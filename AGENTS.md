# AGENTS.md

Guidance for AI coding agents (Claude Code, Aider, Cursor, etc.) working in this repo.

## Project

**ailiance gateway** — OpenAI-compatible router for 11 LLM backends + 13 hardware-specialist LoRA experts (mascarade). EU-sovereign deployment on a home cluster, behind `https://gateway.ailiance.fr` (Cloudflare Tunnel, no auth).

## Tech stack

- Language: Python ≥3.13 (project pinned to 3.13/3.14)
- Runtime: `uv` (never pip / poetry)
- Web: FastAPI + uvicorn
- Test: pytest
- Router model: `sentence-transformers/all-MiniLM-L6-v2` (384d) + MLP head 256→47
- Workers: MLX (Apple Silicon) + llama.cpp (CUDA)

## Conventions

- **Commits**: subject ≤50 chars, no underscore in scope (hooks enforce), no AI attribution by default. End with `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` if generated by Claude and the user asks for attribution.
- **Branches**: `feat/<name>`, `fix/<name>`, `docs/<name>`, `chore/<name>`.
- **PRs**: critic-review mandatory before ship-impacting merges (see `~/.claude/projects/-Users-electron/memory/feedback_critic_before_ship.md`).
- **Tests**: `uv run python -m pytest` — target ≥80 % coverage on new code.
- **Style**: ruff + mypy strict where configured.
- **No `--no-verify`**, no `--admin` on `gh pr merge`.

## File layout (top level)

- `src/gateway/` — FastAPI app, dispatch, FIFO `asyncio.Lock` per `worker_url`, `inference_defaults.py` registry, `tenant_isolation.py`, `alias_inventory.py`, `observability.py`.
- `src/router/` — MiniLM + MLP classifier (router v7), `domain_map.py`, L1+L2 caches.
- `src/worker/` — one model per process, `QUARANTINED_DOMAINS` guard.
- `tools/router_v7/` — multi-LLM corpus generators (PR #77).
- `vendored/iact-bench/` — submodule, real validators (v0.2.0).
- `configs/` — `gateway.yaml`, `gemma4.yaml`, `eurollm.yaml`, `chain_policies.yaml`, plus legacy `apertus.yaml` / `devstral.yaml`.
- `docs/transparency/` — training-data + corpus provenance.
- `tests/` — pytest unit + integration.

## Domain-specific gotchas

- The active systemd unit is **`ailiance-gateway.service`** (NOT `eu-kiki-gateway.service` — that name is legacy, kept `disabled+inactive` for rollback). Cutover happened 2026-05-11.
- The **local clone path is still `~/Documents/Projets/eu-kiki/`** (legacy). The upstream remote is `git@github.com:L-electron-Rare/ailiance.git` (fetch points at the renamed repo). A clone-path rename to `~/Documents/Projets/ailiance/` is documented but **out of scope**.
- The production filesystem on electron-server moved to `/home/electron/ailiance/` (was `/home/electron/eu-kiki/` pre-2026-05-11).
- **Mascarade override** (PR #100, 2026-05-18): 9 hardware domains (`kicad, stm32, emc, embedded, platformio, freecad, dsp, iot, power`) route to MacStudio MLX bf16 `:9340`, NOT to Apertus/Mistral-Medium. The Tower-Ollama Q4 stack on `:8004` is legacy rollback only.
- **`spice` is NOT in `MASCARADE_DOMAINS`** since 2026-05-11 (PR #55) — bench shows −25 on spice-sim. It falls back to `ailiance-apertus` (= Mistral-Medium since the Apertus source was deleted 2026-05-12).
- **`kicad-dsl` and `kicad-pcb`** are overridden to macm1 `:8502` (Gemma-4 + eukiki LoRA, PR #54).
- When adding a new backend alias, also add an entry in `src/gateway/inference_defaults.py` (caller-wins per-alias `temperature`, `max_tokens`, `top_p`, `stop`, `chat_template_kwargs`).
- A request with a `tools[]` field is **force-routed to `qwen-32b-awq`** regardless of classifier output (Mistral-Medium 128B has no native function-calling and hallucinates XML otherwise).
- Reasoning models (`ailiance-reasoning-r1`, `ailiance-ministral-reasoning`) need `max_tokens ≥ 2048` or the chain-of-thought is truncated.
- Qwen3.5 family (`ailiance-qwen`): pass `chat_template_kwargs.enable_thinking=false` for short outputs (scoring, classification).
- Per-worker `asyncio.Lock` is **FIFO** (PR #68 `9440122`) — do not add a new dispatch path that bypasses it without explicit review.
